Jar2UML

Jar2UML imports Java jar files as a UML model into the workspace. It is used to reverse engineer Java API class libraries for PlatformKit‘s Java platform model. Jar2UML is included in the Eclipse MoDisco Toolbox.

Contact

  • Dennis Wagelaar

Download

To download the PlatformKit Eclipse plugin, do the following in your Eclipse:

  • Select “Help → Software Updates → Find and Install...” from the main menu.
  • Select “Search for new features to install” and click “Next”.
  • Click “New Remote Site...”.
  • Choose a name, fill in http://ssel.vub.ac.be/tools/jar2uml/update as URL and click “OK”.
  • Click “Finish”.
  • Select the newly created update site and click “Next”.
  • Now proceed with the installation as you normally would.

Please report any bugs you find in the Jar2UML bugzilla. The source code is available here: http://ssel.vub.ac.be/viewcvs/viewcvs.py/JarToUML/

Documentation

Current Jar2UML documentation is limited to this webpage. For now, a few usage hints will be given below:

  • Start by selecting “File → Import...” in the Eclipse menu.
  • Select “Jar2UML Import → Import Jar Files to UML Models” and click “Next”. The Jar2UML Import Wizard dialog appears (see Fig.1: Jar2UML Import Wizard).
  • At “Select File:”, select the jar files you want te reverse engineer to UML.
  • At “Enter or select the parent folder”, select the folder in the workspace where you want to save your UML model.
  • At “New File Name:”, choose a file name for your UML model.
  • If you only want to import packages starting with “java”, “org.omg”, “org.ietf”, “org.w3c” or “org.xml”, then select “Only Java API packages”. By default, only named public and protected elements are imported. Alternatively, you can select to import private/anonymous elements as well. In addition, you can select to include elements that are referenced by method bytecode instructions.
  • Click “Finish” to create the UML model from the selected jar files.
