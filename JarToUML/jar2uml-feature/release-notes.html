Release Notes

JarToUML 1.1.8 ($Rev$ - $Date$)

  • Improved progress monitor granularity.
  • Added support for method parameter names.
  • Fixed bug 58: Inference of public/protected visibility of invoked virtual methods is inaccurate. This involved the addition of a JVM execution frame simulator to find out the access context of instance fields/methods.
  • Fixed usage of UML's isLeaf().
  • Fixed bug 57: Extract dependencies does not always correctly mark contained vs. inferred elements.

JarToUML 1.1.7 (Rev: 8175 - Date: 2010-07-16 15:22:32 +0200 (vr, 16 jul 2010))

  • Fixed bug 56: Container types of referenced types are removed during dependency model extraction, resulting in dangling references.
  • Fixed bug 55: Package-private is not covered by Jar2UML.
  • Fixed bug 54: Jar file entries with illegal package names (and duplicate class entries) should be ignored.
  • Solved warnings found by FindBugs.

JarToUML 1.1.6 (Rev: 8163 - Date: 2010-07-14 10:55:40 +0200 (Wed, 14 Jul 2010))

  • Fine-tune Java API filter.

JarToUML 1.1.5 (Rev: 8060 - Date: 2010-03-11 15:11:04 +0100 (Thu, 11 Mar 2010))

  • Update test to cover bug 53: JarToUML sometimes processes the same jar/path twice.
  • Fixed bug 53: JarToUML sometimes processes the same jar/path twice.

JarToUML 1.1.4 (Rev: 8056 - Date: 2010-03-11 11:39:12 +0100 (Thu, 11 Mar 2010))

  • Fixed bug 52: Importing Jar/Java project dependencies does not mark derived classifiers (ending with "[]") as inferred.

JarToUML 1.1.3 (Rev: 8049 - Date: 2010-03-08 13:17:25 +0100 (ma, 08 mrt 2010))

  • Refactored code to:
    • Use singleton JarToUML.logger instance.
    • Divide JarToUML steps/passes into separate classes.
    • Guarantee proper type-to-classifier conversion.
  • Added support for support for non-primary jar files (or classpath entries) when importing dependencies.
  • Added unit testing code.
  • Fixed bug: Don't consider a classifier as a derived classifier (with "[]" at the end) of itself.
  • Fixed bug: Array types (inferred, "[]") can also be inferred as Classes, not just DataTypes, e.g. when a clone() method is invoked on them.
  • Fixed bug 50: Import dependencies does not work properly in the presence of cyclic dependencies.
  • Fixed bug 49: Import from Java project fails when a library on the classpath is not a jar file.

JarToUML 1.1.2 (Rev: 7986 - Date: 2010-02-09 18:37:29 +0100 (di, 09 feb 2010))

  • Added support for cyclic dependencies.
  • Jar2UML now parses all nested zip/jar/war/ear/sar/rar files. It does not check if the structure for the special war/ear/sar/rar files is correct.
  • Jar2UML now allows for configuration of log level.

JarToUML 1.1.1 (Rev: 7960 - Date: 2010-02-02 16:20:00 +0100 (di, 02 feb 2010))

  • Set up support for internationalisation (translation).
  • Added custom wizard image.
  • Supports adding inferred element information.
  • Switch to official BCEL plug-in from Eclipse Orbit.

JarToUML 1.1.0 (Rev: 7860 - Date: 2010-01-21 15:12:08 +0100 (Thu, 21 Jan 2010))

  • Switch documentation intro page to direct link to Jar2UML website.
  • Don't save model by default, but make it available via JarToUML#getModel().
  • Generate version info comment for each UML model.
  • Add support for generating UML models from Java projects.
  • Adds annotation with class file format and whether or not any preverified code has been detected (J2ME CLDC).
  • Allow overwriting of existing files by default.
  • Updated comments & Javadoc documentation.

JarToUML 1.0.12 (Rev: 7512 - Date: 2008-08-18 14:16:35 +0200 (Mon, 18 Aug 2008))
